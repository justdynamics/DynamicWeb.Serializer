    The injector MUST use `OpenDialogAction.To<DeserializeFromZipScreen>()` (per D-19) to open the confirmation screen. Do NOT use `ExecuteCommandAction` -- the D-19 flow requires showing dry-run results before the user confirms execution.

    ```csharp
    namespace DynamicWeb.Serializer.AdminUI.Injectors;

    using Dynamicweb.CoreUI;
    using Dynamicweb.CoreUI.Actions;
    using Dynamicweb.CoreUI.Actions.Implementations;
    using Dynamicweb.CoreUI.Icons;
    using Dynamicweb.CoreUI.Screens;
    using Dynamicweb.Files.UI.Screens.Files;
    using DynamicWeb.Serializer.AdminUI.Queries;
    using DynamicWeb.Serializer.Configuration;

    public sealed class SerializerFileOverviewInjector : ScreenInjector<FileOverviewScreen>
    {
        public override void OnAfter(FileOverviewScreen screen, UiComponentBase content)
        {
            var model = Screen?.Model;
            if (model is null)
                return;

            if (!string.Equals(model.Extension, ".zip", StringComparison.OrdinalIgnoreCase))
                return;

            if (!IsInOutputDirectory(model.FilePath))
                return;

            content.TryGet<ScreenLayout>(out var screenLayout);
            screenLayout?.ContextActionGroups.Add(new ActionGroup
            {
                Nodes = [
                    new ActionNode
                    {
                        Name = "Import to database",
                        Icon = Icon.Upload,
                        NodeAction = OpenDialogAction.To<DeserializeFromZipScreen>()
                            .With(new DeserializeFromZipQuery { FilePath = model.FilePath })
                    }
                ]
            });
        }

        internal static bool IsInOutputDirectory(string? filePath)
        {
            if (string.IsNullOrEmpty(filePath))
                return false;

            var configPath = ConfigPathResolver.FindConfigFile();
            if (configPath == null)
                return false;

            var config = ConfigLoader.Load(configPath);
            var outputDir = config.OutputDirectory?.Replace('\\', '/').TrimEnd('/') ?? "";
            var normalizedPath = filePath.Replace('\\', '/');

            return normalizedPath.Contains(outputDir, StringComparison.OrdinalIgnoreCase);
        }
    }
    ```

    **D-19 confirmation flow (MANDATORY per locked decision):**
    The user clicks "Import to database" -> `OpenDialogAction.To<DeserializeFromZipScreen>()` opens the screen -> screen auto-runs dry-run on load -> shows per-table breakdown (each predicate: N new, N updated, N skipped) -> user clicks Confirm -> command executes actual deserialization -> redirects to log viewer.

    **Create `DeserializeFromZipQuery.cs`:**
    Query class with `FilePath` property, passed from the injector's `OpenDialogAction.With()`.

    **Create `DeserializeFromZipModel.cs`:**
    Model with properties:
    - `FilePath` (string) -- the zip file path
    - `DryRunSummary` (LogFileSummary) -- populated on screen load from dry-run results
    - `IsValid` (bool) -- whether the zip contains valid serialization data
    - `ValidationError` (string?) -- error message if zip is invalid (D-21)

    **Create `DeserializeFromZipScreen.cs`:**
    A dialog screen (extend the appropriate DW dialog base class -- check `DialogScreenBase` or similar in DW10 source).

    On load (query handler / `GetModel`):
    1. Load config via `ConfigPathResolver.FindConfigFile()` + `ConfigLoader.Load()`
    2. Extract zip to temp directory via `ZipFile.ExtractToDirectory()`
    3. Validate extracted content has YAML files matching configured predicates (per D-21). If not, set `IsValid = false` and `ValidationError = "This zip doesn't contain valid serialization data. Expected YAML files matching configured predicates."`
    4. If valid, run dry-run: `orchestrator.DeserializeAll(config.Predicates, tempDir, log, isDryRun: true)`
    5. Build `DryRunSummary` from the dry-run `OrchestratorResult`, populating per-predicate Created/Updated/Skipped counts
    6. Clean up the temp directory after dry-run (it will be re-extracted on confirm)
    7. Render the screen showing the per-table breakdown and a "Confirm" button

    The screen layout should display:
    - File name being imported
    - Table with columns: Predicate | Table | New | Updated | Skipped
    - One row per predicate from the dry-run summary
    - If dry-run found errors, show them as warnings
    - "Confirm" button that submits `DeserializeFromZipCommand`
    - "Cancel" button that closes the dialog

    **Create `DeserializeFromZipCommand.cs`:**
    A `CommandBase` that performs the ACTUAL deserialization (called only after user confirms):

    1. Load config via `ConfigPathResolver.FindConfigFile()` + `ConfigLoader.Load()`
    2. Validate the zip file exists and is a .zip
    3. Extract zip to a temp directory via `ZipFile.ExtractToDirectory()` (using `System.IO.Compression`)
    4. Validate the extracted content has YAML files matching configured predicates (per D-21). If not, clean up temp dir and return error: "This zip doesn't contain valid serialization data. Expected YAML files matching configured predicates."
    5. Create orchestrator via `ProviderRegistry.CreateOrchestrator(filesRoot)`
    6. Run actual deserialization: `orchestrator.DeserializeAll(config.Predicates, tempDir, log, isDryRun: false)`
    7. Build `LogFileSummary` from result, generate advice via `AdviceGenerator.GenerateAdvice(result)`
    8. Write per-run log file via `LogFileWriter.CreateLogFile(paths.Log, "DeserializeZip")` + `LogFileWriter.WriteSummaryHeader()` + flush log lines
    9. Clean up temp directory in a finally block (per D-20)
    10. On success, redirect to log viewer: return a `CommandResult` that navigates to `LogViewerScreen` with the new log file selected (per D-19 "redirect to log viewer")
