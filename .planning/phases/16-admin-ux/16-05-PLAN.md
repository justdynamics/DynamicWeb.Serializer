  <read_first>src/DynamicWeb.Serializer/AdminUI/Models/DeserializeFromZipModel.cs, src/DynamicWeb.Serializer/AdminUI/Commands/DeserializeFromZipCommand.cs, src/DynamicWeb.Serializer/Configuration/ConfigPathResolver.cs</read_first>

  <action>
The bug: ConfigPathResolver.FindConfigFile() returns a path like `C:\...\wwwroot\Files\Serializer.config.json`. GetDirectoryName gives `filesRoot = C:\...\wwwroot\Files`. The DW virtual path from the file browser is `/Files/System/Serializer/Download/file.zip`. Combining these produces `C:\...\wwwroot\Files\Files\System\...` -- doubled "Files" segment.

Fix both files identically: use the PARENT of filesRoot (the wwwroot directory) as the base for path resolution, since DW virtual paths already include the `/Files/` prefix.

**In DeserializeFromZipModel.cs** (line 49-50), change:
```csharp
var filesRoot = Path.GetDirectoryName(configPath)!;
var physicalZipPath = Path.Combine(filesRoot, filePath.TrimStart('/', '\\'));
```
To:
```csharp
var filesRoot = Path.GetDirectoryName(configPath)!;
var webRoot = Directory.GetParent(filesRoot)?.FullName ?? filesRoot;
var physicalZipPath = Path.Combine(webRoot, filePath.TrimStart('/', '\\'));
```

**In DeserializeFromZipCommand.cs** (line 44, 49), change:
```csharp
var filesRoot = Path.GetDirectoryName(configPath)!;
...
var physicalZipPath = Path.Combine(filesRoot, FilePath.TrimStart('/', '\\'));
```
To:
```csharp
var filesRoot = Path.GetDirectoryName(configPath)!;
var webRoot = Directory.GetParent(filesRoot)?.FullName ?? filesRoot;
...
var physicalZipPath = Path.Combine(webRoot, FilePath.TrimStart('/', '\\'));
```

IMPORTANT: The `filesRoot` variable is still used elsewhere in both files (for systemDir, orchestrator creation, etc.) -- do NOT change those usages. Only change the physicalZipPath construction to use webRoot.

The orchestrator still needs filesRoot (the Files directory) for its internal operations. Only the zip path resolution needs webRoot.
  </action>
