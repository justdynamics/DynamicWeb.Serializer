    - PrettyPrint("&lt;broken&gt;") with malformed XML returns original string unchanged
    - PrettyPrint('&lt;?xml version="1.0" encoding="utf-8"?&gt;&lt;Settings&gt;&lt;param name="X"&gt;1&lt;/param&gt;&lt;/Settings&gt;') produces multi-line indented XML starting with the declaration line, each child element on its own line, with LF-only line endings (no \r)
    - PrettyPrint('&lt;Settings&gt;&lt;param&gt;1&lt;/param&gt;&lt;/Settings&gt;') (no declaration) produces indented XML WITHOUT a declaration line
    - PrettyPrint output never contains \r characters (CRLF normalized to LF)
    - Compact(null) returns null
    - Compact("") returns ""
    - Compact("not xml") returns "not xml"
    - Compact(prettyPrintedXml) returns single-line XML with no whitespace between elements
    - Compact preserves XML declaration when present
    - Round-trip: Compact(PrettyPrint(original)) produces XML that is semantically identical to original (parseable to same XDocument)
    - Round-trip with declaration: input with &lt;?xml...?&gt; survives PrettyPrint then Compact with declaration intact
  </behavior>

    3. Check if original starts with "&lt;?xml" (case-insensitive trim) to detect declaration

    4. Build result: if had declaration, prepend `xdoc.Declaration?.ToString()` then `xdoc.ToString(SaveOptions.DisableFormatting)`; else just `xdoc.ToString(SaveOptions.DisableFormatting)`
    5. Return result (no newlines in compact output)
