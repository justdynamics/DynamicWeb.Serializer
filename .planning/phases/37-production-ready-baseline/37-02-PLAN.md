<objective>
Unify the schema-tolerance + type-coercion logic that currently exists in two places — `ContentDeserializer` (Area-only, landed in commit `f0bfbba`) and `SqlTableProvider.CoerceRowTypes` (all SqlTable predicates) — into a single shared `TargetSchemaCache` infrastructure class. Apply the unified helper to every raw-SQL write path.

**Scope note:** The phrase "Raw-SQL write paths for Page, Paragraph, GridRow, and ItemType_* tables" in the ROADMAP requirement was imprecise. Code audit shows `ContentDeserializer.cs` only has raw-SQL writes to the `Area` table (lines 452, 491); Page/Paragraph/GridRow/ItemType writes go through DW C# APIs (`Services.Pages.SavePage`, `Services.Items.Save`) which tolerate schema drift at the library level. The real fragility is in (a) ContentDeserializer's Area path (already fixed by f0bfbba, needs extraction) and (b) SqlTableProvider's coercion + implicit column filtering (works today but via a duplicate code path). This plan consolidates.

**Out of scope:**
- Adding schema tolerance to ItemService / SavePage paths — those are DW library internals, not our code
- Schema DDL auto-migration (separate v0.6.0 concern)
</objective>

    ```csharp
    using System.Globalization;
    using Dynamicweb.Data;

    namespace DynamicWeb.Serializer.Infrastructure;

    /// <summary>
    /// Per-run cache of target table schema (columns + SQL data types) plus unified YAML → .NET
    /// type coercion. One instance per deserialize run — shared across ContentDeserializer (Area
    /// write path) and SqlTableProvider (MERGE path). Issues at most one INFORMATION_SCHEMA query
    /// per table accessed. Extracted from commit f0bfbba's GetTargetAreaColumns + CoerceForColumn.
    /// </summary>
    public class TargetSchemaCache
    {
        private readonly Dictionary<string, HashSet<string>> _columns =
            new(StringComparer.OrdinalIgnoreCase);
        private readonly Dictionary<string, Dictionary<string, string>> _types =
            new(StringComparer.OrdinalIgnoreCase);
        private readonly HashSet<(string table, string column)> _loggedMissing = new();
        private readonly Func<string, (HashSet<string> cols, Dictionary<string, string> types)> _schemaLoader;

        /// <summary>
        /// Production constructor: queries INFORMATION_SCHEMA.COLUMNS via Dynamicweb.Data.Database.
        /// Test constructor: pass a loader that returns fixture schema for the supplied table name.
        /// </summary>
        public TargetSchemaCache(Func<string, (HashSet<string> cols, Dictionary<string, string> types)>? schemaLoader = null)
        {
            _schemaLoader = schemaLoader ?? DefaultLoader;
        }

        private static (HashSet<string>, Dictionary<string, string>) DefaultLoader(string tableName)
        {
            var cols = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
            var types = new Dictionary<string, string>(StringComparer.OrdinalIgnoreCase);
            var cb = new CommandBuilder();
            cb.Add("SELECT COLUMN_NAME, DATA_TYPE FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = {0}", tableName);
            using var reader = Database.CreateDataReader(cb);
            while (reader.Read())
            {
                cols.Add(reader.GetString(0));
                types[reader.GetString(0)] = reader.GetString(1);
            }
            return (cols, types);
        }

        public IReadOnlySet<string> GetColumns(string tableName)
        {
            Ensure(tableName);
            return _columns[tableName];
        }

        public IReadOnlyDictionary<string, string> GetColumnTypes(string tableName)
        {
            Ensure(tableName);
            return _types[tableName];
        }

        public object? Coerce(string tableName, string columnName, object? value)
        {
            if (value is null || value is DBNull) return DBNull.Value;
            Ensure(tableName);
            if (!_types[tableName].TryGetValue(columnName, out var dataType)) return value;

            if (value is string s)
            {
                // Empty-string to null for non-string types (preserves f0bfbba + SqlTableProvider current behavior)
                if (string.IsNullOrWhiteSpace(s) && !IsStringType(dataType)) return DBNull.Value;

                switch (dataType.ToLowerInvariant())
                {
                    case "datetime":
                    case "datetime2":
                    case "smalldatetime":
                    case "date":
                        if (DateTime.TryParse(s, CultureInfo.InvariantCulture,
                                DateTimeStyles.AssumeUniversal | DateTimeStyles.AdjustToUniversal,
                                out var dt)) return dt;
                        break;
                    case "datetimeoffset":
                        if (DateTimeOffset.TryParse(s, CultureInfo.InvariantCulture,
                                DateTimeStyles.RoundtripKind, out var dto)) return dto;
                        break;
                    case "bit":
                        if (s.Equals("true", StringComparison.OrdinalIgnoreCase) || s == "1") return true;
                        if (s.Equals("false", StringComparison.OrdinalIgnoreCase) || s == "0") return false;
                        break;
                    case "int":
                    case "smallint":
                    case "tinyint":
                        if (int.TryParse(s, CultureInfo.InvariantCulture, out var i)) return i;
                        break;
                    case "bigint":
                        if (long.TryParse(s, CultureInfo.InvariantCulture, out var l)) return l;
                        break;
                    case "decimal":
                    case "numeric":
                    case "money":
                    case "smallmoney":
                        if (decimal.TryParse(s, CultureInfo.InvariantCulture, out var dec)) return dec;
                        break;
                    case "float":
                        if (double.TryParse(s, CultureInfo.InvariantCulture, out var f)) return f;
                        break;
                    case "real":
                        if (float.TryParse(s, CultureInfo.InvariantCulture, out var r)) return r;
                        break;
                    case "uniqueidentifier":
                        if (Guid.TryParse(s, out var g)) return g;
                        break;
                }
                return value;
            }

            // Non-string incoming value: handle int→long for bigint (carried over from SqlTableProvider)
            if (dataType.Equals("bigint", StringComparison.OrdinalIgnoreCase) && value is int intVal)
                return (long)intVal;

            return value;
        }

        /// <summary>
        /// First time this (table, column) pair is seen, log the warning and return true.
        /// Subsequent calls are silent and return false. Used to avoid log spam when dozens
        /// of rows share the same missing column.
        /// </summary>
        public bool LogMissingColumnOnce(string tableName, string columnName, Action<string>? log)
        {
            if (!_loggedMissing.Add((tableName, columnName))) return false;
            log?.Invoke($"WARNING: source column [{tableName}].[{columnName}] not present on target schema — skipping");
            return true;
        }

        private static bool IsStringType(string sqlType) =>
            sqlType.ToLowerInvariant() is "nvarchar" or "varchar" or "nchar" or "char"
                or "ntext" or "text" or "xml";

        private void Ensure(string tableName)
        {
            if (_columns.ContainsKey(tableName)) return;
            var (cols, types) = _schemaLoader(tableName);
            _columns[tableName] = cols;
            _types[tableName] = types;
        }
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| INFORMATION_SCHEMA query input | The `tableName` parameter passed into TargetSchemaCache comes from predicate config — same trust level as the table name used in the SqlTableWriter MERGE. |
| Coercion input | Row values come from YAML — arbitrary user content (whatever the source DB had). Must not cause SQL injection or DoS during coercion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-02-01 | Tampering (SQL injection via tableName) | TargetSchemaCache.DefaultLoader | mitigate | The `DefaultLoader` uses `cb.Add("... TABLE_NAME = {0}", tableName)` — CommandBuilder parameterizes the value. Unit test: verify that a tableName containing `'; DROP TABLE X;--` does not issue a drop command (the fixture-loader path can verify the parameterized SQL shape). |
| T-37-02-02 | DoS | Coerce with pathological input | accept | YAML deserializer bounds input size; Coerce's regex-free `TryParse` paths are O(n) linear on string length. Not a new surface — SqlTableProvider's existing CoerceRowTypes has the same input characteristics. |
| T-37-02-03 | Information Disclosure | INFORMATION_SCHEMA.COLUMNS leaks schema | accept | The caller already has direct DB access (this is admin tooling on the same host as the DB). The cache only re-exposes schema the caller can already enumerate. |
| T-37-02-04 | Elevation of Privilege | TargetSchemaCache bypassing column filtering | mitigate | Post-refactor, ALL raw-SQL writes that touched schema-sensitive tables pass through `_schemaCache.GetColumns` filter. The regression test `WriteAreaProperties_TargetMissingColumn_Skipped` proves column absence causes exclusion. Code-review check: `git diff` must show no raw SQL UPDATE/INSERT composing column names from YAML data without the filter guard. |
