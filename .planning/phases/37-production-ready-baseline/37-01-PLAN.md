<objective>
Bifurcate the serializer config and run pipeline into two distinct modes — **Deploy** (default, source-wins, ships deployment data like shop structure / currencies / payment methods) and **Seed** (explicit opt-in, destination-wins, ships seed content like Customer Center pages / FAQ copy without overwriting customer edits on re-deploy). The split lives at config *structure* level, not as a per-predicate field (per D-01). After every serialize run, a per-mode files-written manifest drives cleanup of stale YAML from prior runs.

This is the headline Phase 37 delivery: F-01 (seed vs deploy) + F-04 (stale output) solved together because they share the same "mode" abstraction.

Requirement reconciliation note: SEED-01 and SEED-02 in ROADMAP.md were originally framed as per-predicate `deserializeMode` enum. Per CONTEXT.md D-01, this phase implements SEED-01/SEED-02 as a *config structural split* — Deploy and Seed each have their own Predicates list and exclusion config. The split IS the mechanism.

**Out of scope for this plan (handled elsewhere):**
- Schema-tolerance broadening (Plan 37-02)
- Exclusion-list curation (Plan 37-03)
- SqlTable `where` clause (Plan 37-03)
- Strict mode (Plan 37-04)
- LINK-02 + template manifest (Plan 37-05)
</objective>

       ```csharp
       public class ManifestWriter
       {
           public record Manifest
           {
               public required string Mode { get; init; }         // "deploy" or "seed"
               public required DateTime WrittenAtUtc { get; init; }
               public required List<string> Files { get; init; }  // paths relative to modeRoot
           }

           public void Write(string modeRoot, string mode, IEnumerable<string> writtenFiles)
           {
               var manifest = new Manifest {
                   Mode = mode,
                   WrittenAtUtc = DateTime.UtcNow,
                   Files = writtenFiles
                       .Select(f => Path.GetRelativePath(modeRoot, f).Replace('\\', '/'))
                       .OrderBy(f => f, StringComparer.OrdinalIgnoreCase)
                       .ToList()
               };
               var path = Path.Combine(modeRoot, $"{mode}-manifest.json");
               File.WriteAllText(path, JsonSerializer.Serialize(manifest, ManifestJsonOptions));
           }

           public Manifest? Read(string modeRoot, string mode)
           {
               var path = Path.Combine(modeRoot, $"{mode}-manifest.json");
               if (!File.Exists(path)) return null;
               return JsonSerializer.Deserialize<Manifest>(File.ReadAllText(path), ManifestJsonOptions);
           }

           private static readonly JsonSerializerOptions ManifestJsonOptions = new()
           {
               WriteIndented = true,
               PropertyNamingPolicy = JsonNamingPolicy.CamelCase
           };
       }
       ```

    2. Create `Infrastructure/ManifestCleaner.cs`:
       ```csharp
       public class ManifestCleaner
       {
           /// <summary>
           /// Deletes every file under modeRoot that is not listed in writtenPaths and is not the manifest JSON itself.
           /// Empty directories after deletion are also removed.
           /// </summary>
           public int CleanStale(string modeRoot, string mode, IEnumerable<string> writtenFiles, Action<string>? log = null)
           {
               if (!Directory.Exists(modeRoot)) return 0;

               var manifestFileName = $"{mode}-manifest.json";
               var writtenSet = new HashSet<string>(
                   writtenFiles.Select(f => Path.GetFullPath(f)),
                   StringComparer.OrdinalIgnoreCase);

               int deleted = 0;
               foreach (var file in Directory.EnumerateFiles(modeRoot, "*", SearchOption.AllDirectories))
               {
                   if (string.Equals(Path.GetFileName(file), manifestFileName, StringComparison.OrdinalIgnoreCase))
                       continue;
                   if (writtenSet.Contains(Path.GetFullPath(file)))
                       continue;
                   File.Delete(file);
                   log?.Invoke($"Cleanup: deleted stale file {Path.GetRelativePath(modeRoot, file)}");
                   deleted++;
               }

               // Remove any now-empty directories
               foreach (var dir in Directory.EnumerateDirectories(modeRoot, "*", SearchOption.AllDirectories)
                   .OrderByDescending(d => d.Length))
               {
                   if (Directory.Exists(dir) && !Directory.EnumerateFileSystemEntries(dir).Any())
                       Directory.Delete(dir);
               }

               return deleted;
           }
       }
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem write scope | `ManifestCleaner.CleanStale` deletes files under `{SerializeRoot}/{mode}/`. It must NOT escape that subtree. |
| Config input | `ConfigLoader.Load` reads Serializer.config.json; malicious JSON could try to set `OutputSubfolder` to traversal paths like `../../..` |
| Mode parameter (CLI / API) | `SerializerSerializeCommand.Mode` comes from HTTP query string or CLI arg — untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01-01 | Tampering / DoS | ManifestCleaner.CleanStale | mitigate | Cleaner confines deletion to `modeRoot` path. Use `Path.GetFullPath` on both `modeRoot` and each candidate file, and reject candidates that don't start with the resolved modeRoot. Unit test: `CleanStale_SymlinkEscapeAttempt_Rejected` exercises a manually-created symlink pointing outside modeRoot and verifies the outer file is NOT deleted. |
| T-37-01-02 | Tampering | ConfigLoader.Load — `OutputSubfolder` from JSON | mitigate | Validate `OutputSubfolder` matches `^[a-zA-Z0-9_-]{1,32}$` — alphanumeric, underscore, dash, bounded length. Reject `..`, `/`, `\`, absolute paths. Fail config load with clear error. Same validation for the mode name derived from `DeploymentMode`. |
| T-37-01-03 | Tampering | SerializerSerializeCommand.Mode | mitigate | Parse Mode string strictly via `Enum.TryParse&lt;DeploymentMode&gt;(ignoreCase: true)`. Reject anything that doesn't exactly match "Deploy" or "Seed" with HTTP 400 and a clear message. No string-to-path interpolation before enum parse succeeds. |
| T-37-01-04 | Information Disclosure | Manifest JSON | accept | Manifest lists relative YAML file paths only. No credentials, no secrets. Written to the same folder the YAML lives in — equivalent exposure. |
| T-37-01-05 | Repudiation | ManifestCleaner deletions | mitigate | Every deletion is logged via the existing `Action&lt;string&gt;? log` channel and aggregated into `LogFileSummary.Advice`. Operator can audit via the log-viewer admin screen. |
| T-37-01-06 | Denial of Service | ConfigLoader legacy-migration infinite loop | mitigate | Loader ONLY migrates-on-load if `Deploy` is absent; the output is Deploy-present, so a subsequent load will not re-migrate. Explicit test `Load_AlreadyMigrated_IdempotentlyLoads` guards this. |
| T-37-01-07 | Elevation of Privilege | Seed-mode skip leaking target data | accept | Seed's `RowExistsInTarget` check issues a parameterized `SELECT 1 FROM [table] WHERE [key] = @v` via the existing SqlTableWriter.RowExistsInTarget (already parameterized). No user-controlled table/column names on that path — they come from predicate config (which Plan 37-03 will further validate). |
