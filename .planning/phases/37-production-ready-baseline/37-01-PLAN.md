<objective>
The serializer becomes safe to run on a live production instance with end-user-edited
content, and baselines can include rows from tables that mix structural data with
tenant-specific contamination (AccessUser roles vs. tenant demo groups).

Add two orthogonal capabilities:

1. **Per-predicate deserialize mode** — `source-wins` (current default), `if-absent`
   (insert-only seed behavior), or `skip` (serialize but never deserialize back; for
   snapshot-only tracking).

2. **SqlTable `where` clause** — enables row-level selection so a predicate can target
   `AccessUser WHERE AccessUserType = 3` (roles) while excluding tenant groups, without
   forcing the developer to choose between the whole table and nothing.

Both changes flow through the full stack: config model → validation → serialize read
path → deserialize write path → admin UI editor.

Also in scope: surface cross-environment page link resolution failures with their source
(addresses F-17 — today the warning is contextless).
</objective>

<tasks>

### Task 1 — Data model

- Add `DeserializeMode` enum: `SourceWins` (default), `IfAbsent`, `Skip`
- Add `DeserializeMode DeserializeMode { get; init; } = DeserializeMode.SourceWins;` to `ProviderPredicateDefinition`
- Add `string? Where { get; init; }` to `ProviderPredicateDefinition`
- Extend admin UI `PredicateEditModel` + editor with a dropdown (Source Wins / Seed-if-Absent / Skip) + `Where` textarea (only shown when ProviderType=SqlTable)

### Task 2 — SQL identifier / where-clause validator

- New `Configuration/SqlIdentifierValidator.cs`:
  - `ValidateTable(string tableName)` — allowlist against `INFORMATION_SCHEMA.TABLES`
  - `ValidateColumn(string table, string column)` — allowlist against `INFORMATION_SCHEMA.COLUMNS`
  - `ValidateWhereClause(string table, string where)` — parse tokens; reject semicolons, `--`, `/*`, `xp_`, `EXEC`, subselects (`SELECT`); every identifier must resolve to a known column
- Call at `SerializerConfiguration.Load(...)` and at `SavePredicateCommand` (admin UI path)
- Cache the schema lookup per load

### Task 3 — Serialize path

- `SqlTableReader` composes `SELECT … FROM {table} WHERE ({predicate.Where})` when Where is set
- Values must remain parameter-safe — the Where clause is literal but identifiers are validated upstream
- Skip the whole predicate at serialize if `DeserializeMode == Skip` (snapshot-only → don't even write YAML)

### Task 4 — Deserialize path

- `SqlTableWriter.Write`: when `DeserializeMode == IfAbsent`:
  - Build `WHERE` clause on natural key (NameColumn or PK from TableMetadata)
  - Execute `SELECT 1 FROM {table} WHERE {keyCol} = @val` — if any row, SKIP (log `Seeded-skip {key} (already present)`)
  - Otherwise INSERT
- `ContentDeserializer.DeserializePage`: when `DeserializeMode == IfAbsent`:
  - `SELECT 1 FROM Page WHERE PageUniqueId = {guid}` — if present, SKIP (log similarly)
  - Otherwise delegate to existing SavePage path
- Both modes: `Skip` → never write, no log noise

### Task 5 — Cross-env link resolution context (F-17)

- `InternalLinkResolver.ResolveLink(...)` currently logs `WARNING: Unresolvable page ID N in link`
- Extend the signature / call sites to carry the origin `(pageGuid, fieldName, linkIndex)` and include in the warning:
  `WARNING: Unresolvable page ID N in link — origin: page {guid} field '{fieldName}' [#{linkIndex}]`
- The origin context is already available at every call site; it's just not passed through
