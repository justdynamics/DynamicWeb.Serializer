<objective>
Close two observability gaps that keep the serializer from being trustworthy in an automated pipeline:

1. **CACHE-01** — Rewrite cache invalidation. Today's AddInManager-backed `ICacheResolver` silently fails for most services (F-10 baseline test: 9 of 10 cache-type lookups returned "Cache type not found, skipping"). Replace with a curated static map: short-name (or full-type-name) → `Action` that invokes the service's typed `ClearCache()` method directly. Unknown names become ERROR at config-load, not silent skips.

2. **STRICT-01 (SEED-001)** — Add `StrictMode`. Every WARNING emitted during a run is accumulated; in strict mode, the end-of-run assertion throws, producing a non-zero exit. Entry-point-aware defaults: API/CLI default ON (CI/CD is the target), admin UI default OFF (interactive experimentation). Both overridable.

The two requirements land together because strict-mode is the *teeth* behind the cache-resolution error — without strict, a failed cache resolution is still just a log line.

**Out of scope:**
- Adding new cache services beyond the curated list — extensions land via PR to DwCacheServiceRegistry
- TemplateAssetManifest integration (Plan 37-05)
- LINK-02 link-resolution strict escalation (Plan 37-05, but strict-mode infrastructure from here is reused)
</objective>

       ```csharp
       namespace DynamicWeb.Serializer.Infrastructure;

       /// <summary>
       /// Per SEED-001 (STRICT-01): routes warnings through a single policy.
       /// Lenient mode: log and continue (current v0.4.x behavior).
       /// Strict mode: log, record, and throw CumulativeStrictModeException at end of run.
       /// Per D-18: unresolved links, missing templates, unresolvable cache names,
       /// permission-map fallbacks, and schema-drift drops all escalate.
       /// </summary>
       public class StrictModeEscalator
       {
           private readonly bool _strict;
           private readonly Action<string>? _log;
           private readonly List<string> _recordedWarnings = new();

           public StrictModeEscalator(bool strict, Action<string>? log)
           {
               _strict = strict;
               _log = log;
           }

           public bool IsStrict => _strict;
           public int WarningCount => _recordedWarnings.Count;

           /// <summary>
           /// Log a warning. In strict mode, record it for the end-of-run assertion.
           /// The message SHOULD include context (which predicate, page GUID, etc.) because
           /// the end-of-run exception surfaces these verbatim.
           /// </summary>
           public void Escalate(string warning)
           {
               _log?.Invoke(warning.StartsWith("WARNING", StringComparison.OrdinalIgnoreCase)
                   ? warning
                   : "WARNING: " + warning);
               if (_strict) _recordedWarnings.Add(warning);
           }

           /// <summary>
           /// In strict mode, throws if any warnings were escalated. Call once at end-of-run.
           /// Never throws in lenient mode.
           /// </summary>
           public void AssertNoWarnings()
           {
               if (!_strict || _recordedWarnings.Count == 0) return;
               throw new CumulativeStrictModeException(_recordedWarnings);
           }

           /// <summary>Null instance for call sites that don't care. Always lenient, never records.</summary>
           public static readonly StrictModeEscalator Null = new(strict: false, log: null);
       }

       public class CumulativeStrictModeException : Exception
       {
           public IReadOnlyList<string> Warnings { get; }
           public CumulativeStrictModeException(IReadOnlyList<string> warnings)
               : base($"Strict mode: {warnings.Count} warning(s) escalated to failure:\n  - " +
                       string.Join("\n  - ", warnings))
           {
               Warnings = warnings;
           }
       }

       /// <summary>
       /// D-16: entry-point-aware default. API/CLI = default ON; admin UI = default OFF.
       /// Precedence: request-parameter > config file > entry-point-default.
       /// </summary>
       public static class StrictModeResolver
       {
           public enum EntryPoint { Cli, Api, AdminUi }

           public static bool Resolve(EntryPoint entryPoint, bool? configValue, bool? requestValue)
           {
               if (requestValue.HasValue) return requestValue.Value;
               if (configValue.HasValue) return configValue.Value;
               return entryPoint switch
               {
                   EntryPoint.Cli => true,       // D-16: CI/CD target, default ON
                   EntryPoint.Api => true,       // D-16: CI/CD target, default ON
                   EntryPoint.AdminUi => false,  // D-16: interactive, default OFF
                   _ => false
               };
           }
       }
       ```

| Boundary | Description |
|----------|-------------|
| StrictMode request parameter (CLI/API/UI) | Untrusted input; controls whether the run throws on warnings. Cannot compromise integrity but could allow an attacker to skip a safety gate. |
| DwCacheServiceRegistry content | Static, compile-time-fixed. No runtime extension. |
| Aggregated warning output (CumulativeStrictModeException) | Contains diagnostic info about the source baseline — e.g. page GUIDs, template names. Surfaces in HTTP 500 response body. |
