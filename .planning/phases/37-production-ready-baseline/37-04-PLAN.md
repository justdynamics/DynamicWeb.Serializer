### Task 1 — DwCacheServiceRegistry

Replace the AddInManager-based resolution (silently fails for most services per
`project_dw_area_cache.md`). Curated static map:

```csharp
public static readonly IReadOnlyDictionary<string, Action> Clears = new Dictionary<string, Action>(StringComparer.OrdinalIgnoreCase)
{
    ["Dynamicweb.Ecommerce.International.CountryService"]  = () => Dynamicweb.Ecommerce.International.CountryService.ClearCache(),
    ["Dynamicweb.Ecommerce.International.CurrencyService"] = () => Dynamicweb.Ecommerce.International.CurrencyService.ClearCache(),
    ["Dynamicweb.Ecommerce.International.LanguageService"] = () => Dynamicweb.Ecommerce.International.LanguageService.ClearCache(),
    ["Dynamicweb.Ecommerce.International.VatGroupService"] = () => Dynamicweb.Ecommerce.International.VatGroupService.ClearCache(),
    ["Dynamicweb.Ecommerce.International.VatGroupCountryRelationService"] = () => Dynamicweb.Ecommerce.International.VatGroupCountryRelationService.ClearCache(),
    ["Dynamicweb.Ecommerce.International.CountryRelationService"] = () => Dynamicweb.Ecommerce.International.CountryRelationService.ClearCache(),
    ["Dynamicweb.Ecommerce.Orders.PaymentService"]  = () => Dynamicweb.Ecommerce.Orders.PaymentService.ClearCache(),
    ["Dynamicweb.Ecommerce.Orders.ShippingService"] = () => Dynamicweb.Ecommerce.Orders.ShippingService.ClearCache(),
    ["Dynamicweb.SystemTools.TranslationLanguageService"] = () => Dynamicweb.SystemTools.TranslationLanguageService.ClearCache(),
    // Direct-ClearCache pattern per project_dw_area_cache memory
};
```

Load-time behavior:

- `SerializerConfiguration.Load` validates every `serviceCaches` entry against this map
- Unknown entries: configuration fails with clear error listing the unknown service
  AND the currently-supported list
- At deserialize time, invocation is just `Clears[name].Invoke()` — no reflection

### Task 2 — TemplateAssetManifest

Two phases:

- **Serialize**: after walking all pages/paragraphs, emit a `templates.manifest.yml`
  under SerializeRoot with entries like:
  ```yaml
  - path: "Swift-v2/Emails/1ColumnEmail.json"
    kind: "grid-row"
    referencedBy: ["page 8410 (Customer Center > Account)"]
  - path: "Swift-v2/Swift-v2_Page.cshtml"
    kind: "page-layout"
    referencedBy: ["page 6869 (Home), page 107 (About), ..."]
  ```
- **Deserialize**: read the manifest first, for each entry check `Files/Templates/Designs/**`
  for existence. Log one WARNING per missing asset with exact filename and list of
  pages that need it.

In strict mode, missing templates are ERRORs.

### Task 3 — Strict mode (SEED-001)

- Add `strictMode: bool` to `SerializerConfiguration`
- Add `--strict` CLI flag and query parameter on `/Admin/Api/SerializerDeserialize`
- Thread through `SerializerOrchestrator.DeserializeAll(strict)` into ContentProvider
  and SqlTableProvider
- Each warning site: if strict, append to a `strictErrors` list AND continue (so all
  warnings are reported at the end, not just the first)
- At end of run: if `strictErrors.Any()`, throw — API returns non-zero HTTP
- Cache invalidation warnings, template missing, unresolvable links, schema-drift
  drops — all become strict errors

### Task 4 — Cross-env link resolution for SqlTable (F-17 / LINK-02)

`InternalLinkResolver`:

- Extract the scan+rewrite logic from the content-specific path
- Add `ResolveInStringColumn(value, sourcePageMap, targetPageMap) -> (rewritten, unresolvedLinks)`
- `SqlTableWriter` calls this for every varchar/nvarchar column on MERGE/INSERT
- Pattern matched: `Default.aspx?ID=N` (and the raw `ID=N` variant inside XML)

Unresolved links: WARNING in default mode, ERROR in strict.
