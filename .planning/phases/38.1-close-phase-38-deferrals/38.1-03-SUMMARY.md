## Decisions Made

- **Decision: Defer Administrator password verification to Step 7 post-host-boot.** Bacpac-restored DBs ship with a known password hash; if it differs, token auth at Step 7 returns 401 and the pipeline throws with a clear pointer to the manual-SSMS reseed fallback in the README. Alternative considered: inline a PBKDF2 hash computation in PowerShell and `UPDATE AccessUser`. Rejected because DW's hash format (iterations, salt shape, column names) is version-specific and brittle — silent fake-success is worse than a loud throw with recovery instructions.
- **Decision: Extract bacpac from zip into the timestamped run dir, then `Get-ChildItem -Recurse -Filter *.bacpac` to find the payload.** Supports both flat and nested zip layouts without hardcoding `swift.bacpac` as the inner filename.
- **Decision: `Stop-HostOnPort` uses `Get-NetTCPConnection` + `taskkill /F /PID` rather than `Stop-Process -Name dotnet`.** The host project uses `dotnet run` which may spawn multiple `dotnet.exe` processes; port-based targeting is precise. `Stop-Process -Name dotnet` would kill unrelated dotnet workloads.
- **Decision: `Invoke-Sqlcmd-File` uses `sqlcmd -b` (batch abort on error).** Ensures non-zero exit code on any T-SQL error so the pipeline throws predictably. Also scans the log output for `ABORT|ROLLBACK\b` as a second gate — scripts 06/07/09 embed those strings in guarded-failure branches (count-mismatch aborts).
- **Decision: Mirror-then-assert pattern for Step 13 SerializeRoot.** Remove prior `deploy/` and `seed/` subdirs specifically (never a blanket `Remove-Item -Recurse` on `$cdSerRoot` or its parent). Avoids the "accidentally deleted a tracked file via git clean in a worktree" class of bugs.
- **Decision: Emit `summary.json` ONLY on full success.** Absence of `summary.json` in a run-dir unambiguously signals a mid-pipeline failure. The thrown error message + per-step `.log` files diagnose cause.
