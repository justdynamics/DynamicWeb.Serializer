## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] Cross-host SerializeRoot mirror step missing from recipe**

- **Found during:** Task 6 Part B step 14 (Deserialize Deploy returning HTTP 400 with SHOP19 FK escalation despite Swift-2.2 source being clean)
- **Issue:** Plan recipe steps 11-13 start hosts and run purge, but don't mirror Swift-2.2's freshly-written YAML baseline to CleanDB's own wwwroot/Files/.../SerializeRoot/. CleanDB host reads its own filesystem, which had STALE Phase-38-05-era YAMLs still containing GROUP253$$SHOP19.yml. Phase 38-03 e2e-results lines 134-136 documents this as "Cross-host SerializeRoot mirror via PowerShell Copy-Item" — Phase 38-03 Deviation 4 — but the Phase 38.1 plan recipe did not propagate the step.
- **Fix:** After Serialize Seed completed, applied `cp -r $SWIFT22_HOST/.../SerializeRoot/{deploy,seed} $CLEANDB_HOST/.../SerializeRoot/`, then re-purged CleanDB and retried Deserialize Deploy. Post-mirror: zero SHOP19 rows in CleanDB EcomShopGroupRelation.
- **Impact:** Exposed the true B.4.1 status: it IS closed (0 orphans post-fix), but without mirror the stale SHOP19 YAML was still re-asserting the orphan. Documented in `38.1-01-e2e-results.md` §"Recipe deviation applied".

**2. [Rule 2 — Missing critical] Operational ALTER TABLE for schema-alignment not in recipe**

- **Found during:** Task 6 Part B step 16 (second Deserialize Deploy attempt after mirror fix; surfaced 3 B.3 Area-column schema-drift escalations)
- **Issue:** Plan Task 3 documents the 7 B.3.1 drift columns in env-bucket.md as "operational: align DW NuGet versions OR config allowlist" — i.e., operator applies ALTER TABLE externally. But the plan's Task 6 E2E recipe does not enumerate the actual ALTER TABLE commands. Without them, both the 3 B.3 Area columns AND the 7 B.3.1 columns escalate strict mode and block HTTP 200.
- **Fix:** Applied 10 ALTER TABLE statements on CleanDB: `Area.AreaHtmlType/AreaLayoutPhone/AreaLayoutTablet` (nvarchar NULL matching Swift-2.2) + `EcomGroups.GroupPageIDRel` + 6 EcomProducts columns (`ProductPeriodId`, `ProductVariantGroupCounter`, `ProductPriceMatrixPeriod`, `ProductOptimizedFor`, `MyVolume`, `MyDouble`). Column types sourced from Swift-2.2's INFORMATION_SCHEMA.COLUMNS.
- **Impact:** After ALTER TABLE, zero B.3/B.3.1 schema-drift escalations in either Deserialize call. B.3.1 demonstrably CLOSED.
- **Documented in:** `38.1-01-e2e-results.md` §"Schema-alignment deviation applied" and within the cleanup-script commentary.

**3. [Rule 3 — Blocking] Auth flow Bearer token (not basic auth)**

- **Found during:** Task 6 Part B step 14 first attempt (HTTP 401 on `-u "Administrator:Administrator1"`)
- **Issue:** Plan's recipe used `-u "admin:<password>"` basic-auth syntax; DW 10.23.9 Admin API requires Bearer token via `/Admin/TokenAuthentication/authenticate` per Phase 38-05 plan.
- **Fix:** Switched curl calls to `POST /Admin/TokenAuthentication/authenticate` to obtain the JWT, then `-H "Authorization: Bearer $TOKEN"` on subsequent API calls.
- **Impact:** Unblocks all 4 API calls. Tokens expire 5min; re-fetched per call for safety.

**4. [Rule 2 — Intentional retain] ALLCAPS `ENVIRONMENT` inside rationale blockquote (D-38.1-17)**
