## Issues Encountered

- **Pipeline's bash → PowerShell exit-code masking:** the `pwsh ... 2>&1 | tee ...` pattern in the invocation masks the PowerShell process exit code because `tee`'s exit code is always 0. Captured by checking `PIPELINE_EXIT` after the pipe (via `$?` / `$LASTEXITCODE`) separately. Pipeline correctly returned 1 (throw in `Invoke-Sqlcmd-File`).
- **SQL session clung briefly post-halt:** after script 08's XACT_ABORT rollback, a `dotnet run --no-build` process lingered briefly (from prior cleanup-script sqlcmd child); port state nevertheless clean (no listeners on 54035/58217), so no retry cleanup needed.
- **ItemTypeDefinitions in-DB is empty.** During root-cause analysis, queried `SELECT COUNT(*) FROM ItemTypeDefinitions` — returned 0. DW loads definitions from XML at runtime; restoring the bacpac doesn't populate this table. Means SQL cannot self-discover link-typed columns; must consult the filesystem XML, which is incomplete (CustomerCenterApp XML absent despite the DB table existing with ~10 link-named columns).
