## Disposition rationale — why scripts 08 and 09 were bypassed

Plan 02 authored scripts 08 (orphan-link-ref cleanup for 20 IDs) and 09 (two misconfigured property pages) against the expected state at Phase 38.1-01's test time (2026-04-21). When the pipeline restored the pristine 2026-01-29 bacpac, direct SQL scans confirmed:

- **Zero occurrences** of `Default.aspx?ID=N` for any of the 20 orphan IDs across all 131 `ItemType_Swift-v2_*` tables, all nvarchar/varchar/ntext/text columns of non-log tables, and the BAK snapshot tables.
- **Zero occurrences** of the 20 IDs as raw integer values in link-type int columns (scan returned only coincidental matches on config fields like `Accordion.Accordion_Items = 2` and `LocationsMap.InitialZoomLevel = 4`).
- **Zero occurrences** of `Default.aspx?ID=<orphan>` in freshly-serialized YAML output from the bacpac state — 50 total `Default.aspx?ID=N` references across 22 distinct valid page IDs (103, 106, 107, 109, 111, 114, 490, 740, 1685, 1840, 1846, 4897, 5862, 6869, 8284, 8285, 8345, 8385, 8388, 8398, 8400, 8431).
- **Host startup delta = 0 rows**: starting the Swift 2.2 host against the bacpac-restored DB did not add any rows to the tracked set (Page, Paragraph, ItemType_Swift-v2_Button, ItemType_Swift-v2_Text, ItemType_Swift-v2_Master, ItemType_Swift-v2_PageProperties). ItemType XML → DB sync at host boot does not introduce orphan references.

**Conclusion:** The 47 orphan warnings Phase 38.1-01 Task 6 Part B observed came from ~3 months of live user edits between the bacpac date (2026-01-29) and the test run (2026-04-21). Those edits — page deletions, button-target changes, etc. — left dangling `Default.aspx?ID=N` references to pages that existed when the button was authored but were later removed. The bacpac itself is clean of this class.

Scripts 08 and 09 remain in `tools/swift22-cleanup/` and in the pipeline chain as **defensive operational tooling**: if a future bacpac snapshot contains such accumulated edits, or if someone runs the round-trip against a non-pristine DB, the scripts will find and clean the references. For a pristine bacpac, they are no-ops by construction.
