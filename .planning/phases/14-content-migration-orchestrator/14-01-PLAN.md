Migrate config loading to unified ProviderPredicateDefinition and create ContentProvider adapter.

    ContentProvider does NOT extend SerializationProviderBase (it doesn't need YAML helpers — ContentSerializer handles that internally). It directly implements ISerializationProvider.

    **Serialize implementation details:**
    - Build `SyncConfiguration` with `OutputDirectory = Path.Combine(outputRoot, "_content")` and `Predicates = [predicate]` (single predicate list)
    - Create `new ContentSerializer(config, log: log)` and call `serializer.Serialize()`
    - Count .yml files in the _content/ directory for the result
    - Return `new SerializeResult { RowsSerialized = fileCount, TableName = "Content" }`
    - Note: ContentSerializer expects `SyncConfiguration.Predicates` to be `List<ProviderPredicateDefinition>` after Task 1

    **Deserialize implementation details:**
    - Build `SyncConfiguration` with `OutputDirectory = Path.Combine(inputRoot, "_content")` and `Predicates = [predicate]`
    - Determine `filesRoot` — this is needed by ContentDeserializer for template resolution. It should be derivable from inputRoot (go up from SerializeRoot to find Files/ root). For the adapter, accept a `filesRoot` in constructor or derive it.
    - Actually, looking at how commands use it: `filesRoot = Path.GetDirectoryName(configPath)` which is the Files/ folder. The orchestrator will need to pass this. Add a `string? filesRoot` constructor parameter to ContentProvider.
    - Create `new ContentDeserializer(config, log: log, isDryRun: isDryRun, filesRoot: _filesRoot)` and call `deserializer.Deserialize()`
    - Map `DeserializeResult` to `ProviderDeserializeResult`: Created, Updated, Skipped, Failed, Errors map directly. Set TableName = "Content".

    **Tests:**
    ContentSerializer/ContentDeserializer depend on DW runtime (Services.Areas, Services.Pages, etc.) so they can't be unit tested directly. Test the adapter's ValidatePredicate logic and verify that the SyncConfiguration is constructed correctly.

    For Serialize/Deserialize tests: use a mock/spy approach.     If mocking is too complex, focus ValidatePredicate tests (which are pure logic) and a structural test that ContentProvider implements ISerializationProvider correctly.
